# アートアセット仕様書（手描き・イラスト調）

ChatGPTで生成した手描き画像を `Assets/Resources/` に置くと、ゲームが自動でその画像を使います。
**画像が無い項目は従来のコード生成（円/四角）で描画される**ので、1枚ずつ差し替えてOKです。
コンパイル・既存挙動は壊れません。

---

## 1. 共通ルール（重要）

- **形式**: PNG / **背景は完全透過**（アルファ付き）
- **キャンバス**: 正方形（例 512×512）。キャラは中央配置、**高さの約85〜90%**を埋める
- **向き**: 正面〜やや見下ろし（トップダウンビュー）。1体のみ、影や地面は描かない
- **色**: 下表の「主な色」を必ず踏襲する（プレイヤーが色で敵を見分けるため。読みやすさ最優先）
- **読み込み設定**: 自動で `filterMode=Bilinear` / 中央ピボット / 高さ=1ワールド単位に正規化されるため、
  解像度は自由（256でも512でも見た目サイズは同じ）。**ボスだけ大きめ解像度推奨（768〜1024）**

### スタイル統一用プレフィックス（全プロンプトの先頭に付ける）
毎回これを先頭に貼ると、全キャラの絵柄が揃います。

```
2Dゲーム用のキャラクター1体を、手描きのダークファンタジー風イラストで描いてください。
なめらかな輪郭線とグラデーション塗り、やわらかい陰影とハイライト。
背景は完全に透明（透過PNG）、影や地面は描かない。キャラを中央に配置し、正方形キャンバスの高さ85〜90%を占める大きさ。正面〜やや見下ろしの構図。
```

> **透過のコツ**: ChatGPTが背景を付けてしまう場合は「背景は単色の白ではなく、完全な透明（アルファ）にしてください。被写体だけを切り抜いた状態で」と念押し。
> それでも背景が残るときは、生成後に背景除去（remove.bg 等）で透過にしてから保存。

## 4. 背景タイル（任意）

地面は既に `Assets/Resources/Tiles/` の grass / soil / stone / floor を自動で使うようになりました
（草原=grass・森=soil・墓地=stone・城=floor）。**今すぐ差し替え不要**です。

テーマに合わせて専用タイルを作りたい場合は、以下の名前で置くと優先的に使われます（**不透明・タイリング可能**な512×512）:

| ファイル | ステージ | プロンプト |
|---|---|---|
| `Tiles/grassland.png` | 草原 | シームレスにタイリングできる草地のテクスチャ。上から見た芝生、自然な色ムラ。継ぎ目が出ないように上下左右がつながること。 |
| `Tiles/forest.png`    | 暗黒の森 | シームレスにタイリングできる暗い森の地面。土・落ち葉・苔。 |
| `Tiles/graveyard.png` | 墓地 | シームレスにタイリングできる古い石畳。苔と亀裂、薄暗い。 |
| `Tiles/castle.png`    | 城 | シームレスにタイリングできる城の石床。整然としたレンガ、暗い色調。 |

> ※背景タイルは透過**不要**（不透明）。シームレス（繰り返しても継ぎ目が出ない）が重要なので、
> プロンプトに必ず「seamless / tileable / 上下左右がつながる」を入れること。ChatGPTは継ぎ目が出やすいので、
> 既存の grass/soil/stone/floor で十分なら無理に作り直さなくてOK。

## 5. 入れ方（Unity）

1. 上記フォルダに PNG をドラッグ＆ドロップ（ファイル名は表のとおり）
2. Unityが自動インポート → そのまま Play で反映
3. 大きさ・位置がズレる場合は調整するので教えてください（コード側の倍率で微調整できます）

## 6. 優先生成順のおすすめ
1. `player.png`（一番目立つ）
2. 草原で出る敵: `chaser.png` `runner.png` `tank.png` `dasher.png`
3. `boss.png`
4. 以降、森→墓地→城 の敵を順次
